# 대국기록 데이터 정리

대국기록은 한 번의 대국 단위 정보와, 해당 대국에 참여한 플레이어별 결과 정보를 함께 저장한다.

## 필수 데이터 항목

| 화면 항목 | DB 컬럼 후보 | 타입 후보 | 입력/계산 | 설명 |
| --- | --- | --- | --- | --- |
| 날짜 | `played_at` | `date` | 입력 | 대국이 진행된 날짜 |
| 구분 | `game_type` | `text` 또는 `enum` | 입력 | 일반전, 스프린트 등 대국 종류 |
| 대국번호 | `match_no` | `integer` | 입력 또는 자동생성 | 날짜 또는 전체 기준 대국 식별 번호 |
| 이름 | `player_name` | `text` | 선택 | 플레이어 표시용 이름 |
| 점수 | `raw_score` | `integer` | 입력 | 대국 종료 후 점봉 원점수 |
| 순위 | `rank` | `integer` | 계산 | 점수 기준 1~4위 |
| 우마 | `uma` | `integer` 또는 `numeric` | 계산 | 순위에 따른 우마 점수 |
| 오카 | `oka` | `integer` 또는 `numeric` | 계산 | 1위에게 적용되는 오카 점수 |
| 최종점수 | 계산값 | `numeric` | 계산 | 점수, 우마, 오카를 반영한 최종 산출 점수 |
| 반영점수 | 계산값 | `numeric` | 계산 | 통계/랭킹에 실제 반영되는 점수 |
| 등록시간 | `created_at` | `timestamptz` | 자동생성 | 기록이 저장된 시간 |
| 등록자 | `created_by` | `uuid` 또는 `text` | 자동/입력 | 기록을 등록한 사용자 |

## 권장 테이블 구조

### `matches`

대국 1건의 공통 정보를 저장한다.

| 컬럼 | 타입 후보 | 설명 |
| --- | --- | --- |
| `id` | `uuid` | 대국 고유 ID |
| `played_at` | `date` | 대국 날짜 |
| `game_type` | `text` | 대국 구분 |
| `match_no` | `integer` | 대국번호 |
| `created_at` | `timestamptz` | 등록시간 |
| `created_by` | `uuid` | 등록자 |

### `match_players`

대국에 참여한 플레이어별 결과를 저장한다. 한 대국당 4개의 row가 생성된다.

| 컬럼 | 타입 후보 | 설명 |
| --- | --- | --- |
| `id` | `uuid` | 대국 플레이어 결과 고유 ID |
| `match_id` | `uuid` | `matches.id` 참조 |
| `player_id` | `uuid` | 플레이어 ID |
| `player_name` | `text` | 기록 당시 플레이어 이름 스냅샷 |
| `raw_score` | `integer` | 입력 점봉 원점수 |
| `rank` | `integer` | 최종 순위 |

## 계산 기준 메모

- 순위는 `raw_score` 내림차순으로 계산한다.
- 동점이 발생할 수 있으므로 동점 처리 기준이 필요하다.
- 우마는 대국 구분 및 배율 설정에 따라 달라질 수 있다.
- 오카는 보통 1위에게만 적용한다.
- 최종점수와 반영점수는 저장된 `raw_score`, 순위, 구분을 기준으로 계산한다.
- 등록자는 Supabase Auth를 사용하면 `auth.users.id`를 `created_by`에 저장하는 방식이 적합하다.

## 화면 표시용 컬럼 순서

